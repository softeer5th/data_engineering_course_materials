# Review

## 강의

### 회고와 리뷰의 차이점?

회고 → 내가 어떻게 하면 더 잘할 수 있었을까? (과정)

리뷰 → 내가 무엇을 했는가? (결과)

1. 내가 무엇을 했다.
2. 무엇을 한 것들에 대한 회고를 한다.

### 회고 잘 하는 법

1일 리뷰 → 회고

- KPT (Keep Problem Try)
    - **`Keep`** 오늘 뭘 했는데 지속해나가자 (잘된 부분들)
    - **`Problem`** 오늘 어떠한 문제가 있어서 다르게 해보자 (오늘 경험한 문제점에 대한 개선)
    - **`Try`** 무엇을 안해봤는데 어떻게 해보자. (이런거 해보면 좋겠다.)

회고의 목적은 과정의 개선에 있다.

오늘 일을 잘했다 못했다가 아니라 방식을 개선하는 것! learning **`process`**에 대한 개선.

# **Retrospective**

## Keep

- 팀원들과 함께 진행상황을 공유하고 같이 성장하는 방향성을 위해서, 데일리 리뷰와 데일리 스크럼을 진행할 것을 제안하였다.
    
    → 이후 팀원들과 함께 공부하고 리뷰하는 좋은 방향성이 설정된 것 같으며, 계속해서 진행할 예정
    
- 공식문서를 찾으면서 진행한 결과, 대강 사용법만 알던 해당 메소드들에 대해서 좀 더 자세한 필드값과 어떠한 의도로 코드가 작성되었는지 확인 할 수 있었다.
    
    → 계속해서 새로운 것들을 공부할 때 공식 문서를 주로 참고하여 공부할 예정이다.
    

## Problem

- GPT로 문제에 대한 해답을 찾아가면서 공부하는 경우, 공부가 잘 진행되지 않은 경험이 있다는 팀원의 피드백을 받았다.
    
    → 다음날 부터는 GPT보다는 공식 문서 자체를 기반으로 공부해보기로 함.
    
- 첫날 진행한 팀 데일리 리뷰여서, 체계적이지 못한 부분이 있었다.
    
    → 다음 회의부터는 먼저 자신의 데일리 리뷰를 작성한 이후에, 해당 문서를 기반으로 함께 팀 데일리 리뷰를 진행해보자.
    

## Try

- 다음부터는 notion에 탬플릿을 만들어서 해당 템플릿을 바탕으로 개인 데일리 리뷰를 작성하자.
